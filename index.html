1. Overview

ASCapstone2025 is a PHP-based web application designed for managing customer profiles, administrative tasks, user accounts, and providing a public-facing homepage for a pool table movers business. The application supports customer management (CRUD operations), admin login and registration, quote requests, and dynamic content display through modular sections. It follows a modular architecture for maintainability and scalability, with separate concerns for views, models, and assets.

Purpose: To provide a centralized system for business operations, including customer data management, admin oversight, and user interaction via forms and displays.

Scope: Includes user authentication, customer profile management, admin panel, public homepage with sections like reviews, services, locations, and a contact form.

2. System Architecture

The application follows an MVC-like pattern, with PHP handling controller and view logic.

Frontend
  • HTML rendered by PHP views.
  • CSS for styling (separate files such as homeStyle.css, adminStyle.css).
  • JavaScript for interactivity (e.g., adminScript.js, script2.js).
  • External libraries: Bootstrap 5.3.3, TinyMCE, Select2, and jQuery.
  • Modular sections via homeViewSections/ (e.g., header.php, reviews.php, form.php, carousel.php, companyDescription.php, footer.php,services.php).
Backend
  • PHP scripts for business logic and data handling.
  • Models in models/: db.php (PDO connection), model_admin.php (admin & customer operations), customer_profiles.php.
  • Database configuration via dbconfig.ini.
  • Session-based authentication for admin access.
Assets
  • Images in images/ (e.g., LogoTemp.png, BilliardBalls.png) and uploaded images in bcimage/.
  • Image file uploads handled by upload.php.
Data Flow
  1. User accesses a page (e.g., homeView.php via index.php redirect).
  2. PHP view processes session, includes models, renders HTML with includes.
  3. Models interact with database using PDO for queries.
  4. JavaScript handles client-side validation and interactions.
  5. Responses update the UI or redirect as needed.

4. Database Design

Configuration: Stored in models/dbconfig.ini (servername, port, dbname, username, password).

Connection: Established in models/db.php using PDO with security-focused attributes.

Tables:

  • customers: ID, firstName, lastName, phoneNumber, email, street, apt, city, state, zipcode, serviceRequested, notes.
  • adminlogin: username, password (hashed with SHA1 + salt), adminEmail.
  • Other tables may exist for reviews, services, etc.

Security: Passwords hashed with SHA1 and a salt (MY-TOP-SECRET-SALT). Supports legacy plaintext login for backward compatibility.

Queries: Prepared statements used to prevent SQL injection.

5. Frontend Components

Main Views:

  • homeView2.php: Public homepage (admin header if logged in) with modular sections.
  • adminView.php: Admin dashboard (manage customers).
  • loginView.php: Admin login form.
  • updateAccount.php / updateAccountView.php: Account update forms.
  • addAdminView.php: Add new admin.
  • privicyPolicy.php: Privacy policy information.

Modular Sections (homeViewSections/): header.php, reviews.php, form.php, carousel.php / carousel2.php, companyDescription.php, footer.php, locations.php, map.php, services.php.

Styling: Separate CSS files per view with Bootstrap for consistency.

Interactivity: JavaScript files for admin operations and form validation.

6. Backend Components

Models (models/):

  • db.php: Database connection setup (PDO).
  • model_admin.php: Authentication, Customer CRUD, Admin management, Quote requests.
  • customer_profiles.php: Additional customer-related functions (assumed).
  • php.php: Contains addition php logic and functions. (assumed).
  • UpdateAccount.php: Functions for update account page
  • updateServices.php: Allows access to tinymce to update services displayed on frontend.

File Uploads: Handled in upload.php

Controllers: PHP view files act as controllers, processing POST/GET, calling model functions, and rendering views.

Security: Input validation, error logging, session checks.

7. Security Considerations

  • Authentication: Session-based admin login. Passwords hashed with SHA1 + salt.
  • Authorization: Admin-only access to privileged views.
  • Input Validation: Required fields validated server-side; client-side via JS.
  • SQL Injection Prevention: PDO prepared statements.
  • File Uploads: Validated type/size in upload.php.
  • Error Handling: Try/catch around PDO; error logging.
  • HTTPS: Recommended for production.
  • Session Management: Standard PHP sessions; consider ID regeneration on login.

8. Deployment

  • Environment: XAMPP on Windows (c:/xampp/htdocs/ASCapstone2025).
  • Requirements: Apache, MySQL, PHP.
  • Setup: Place files in htdocs, configure dbconfig.ini with DB credentials.
  • Production: Dedicated server, environment variables for config, enable HTTPS, database backups.

9. Extensibility & Future Improvements

  • Modular additions to homeViewSections/ or new models.
  • Potential RESTful API for mobile apps.
  • Frontend SPA (React/Vue) and accessibility improvements.
  • Backend roles/permissions; possible OAuth.
  • Testing: Unit tests for models; integration tests for views.
  • Performance: Query optimization; asset caching.
  • Monitoring: Action logging; error tracking.
  • Scalability: Consider Laravel framework for stronger structure.
